Connect 4 java project

As a part of my computer sciance A-Level I had to make a java project call the "NEA" (no idea what that stands for). The NEA is worth 20% of my final grade so it's esental to work hard on it and get a good grade for it. Thankfully i did the majory of the coding durning my summer holiday leaving me to gradualy finish it over my final year without a final rush to finish it.

This is my larges project to date with +300hrs of work that has gone in coding this. However it is my first full size coding project so i am sure if i was to redo it it would only take me 50hrs or so, but hey i enjoyed every minet of it so i'm not complaining.

The AIM:

I had the flexibily to make any project i wanted within reason (Thank you teacher). Chosing the right project is very important for somthing that's supost to show your ability. Too hard and you don't finish, but too easy it's too simple. In the end I chose Connect 4 beacuse wiht this game i can create the simple game easly but then i can make an AI oponent that can be as complex as i have time for.

There was quite o few hoops i had to jump though because it was a school project but to make this a fun story i am just going to talk about the interst coding and the funny bugs that happeren.

(I wanted to do a 2D light ray tracer which i still would like to do in the future but not a good thing for my first project)

Part 1, Making The Base game:

If you don't know what Connect 4 is then fine i will explain:

Connect 4 is two player game where each player takes it in turn to drop there counter in to the vertical board. The aim of the game is to get 4 counter in a row (horisontal, vertical or diagonaly)

Time To Start Coding

To start things off I used a 2D array of strings as the game board data type so i could easly print the board to the console. I then wrote a simple algorthm for placing the counter in the lowest posible (To emulate gravity). Did it work first time? Ovcore not but with a nice helping of debuging the simple game was working. The game didn't know if someone wins and crashed if you try to do anything not intended, But that is fine at this point.

Part 2, Making The Game Smart

First off i needed to know if a player has won or not. So I write an algorthm that looks at every counter and every direction around it to see if there is 4 counters in a row. This is the algorthm i come up with:

for every column on the board
	for: every counter in this column
		store the "counter type" at this position
		for: every positive direction
			start a "chain length" counter
			while: this counter type = "counter type"
				step forward in that direction on board
				if: this counter type = "counter type"
					increase the counter
				else:
					break out of while loop
			move back to starting position
			invert the direction
			while: this counter type = "counter type"
				step forward in that direction on board
				if: this counter type = "counter type"
					increase the counter
				else:
					break out of while loop
			if: "chain length" >= 4
				return "counter type"
return null
